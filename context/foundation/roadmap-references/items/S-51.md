# S-51: Blocked task status

- **Outcome:** user can mark a started task as blocked — waiting on someone or something — from the task list, at session end, and during a running cycle alongside "Gotowe"; blocking mid-cycle hands the session off to a break exactly as completing the task does
- **Change ID:** blocked-task-status
- **Linear:** —
- **GitHub:** —
- **PRD refs:** US-06
- **Prerequisites:** F-07, S-45, S-50
- **Parallel with:** S-52 (independent — no shared surface beyond the task card)
- **Blockers:** —
- **Acceptance:**
  - Blocked is a fourth task state next to active, planned, and completed — visible as such in the task list, not folded into completion or the stale archive
  - User can mark a task blocked from the task list (Zadania)
  - User can mark the focused task blocked at session end, in the same closure surface where the session is wrapped up
  - User can mark the focused task blocked during a running cycle, from the same place "Gotowe" is offered
  - Blocking during a running cycle ends the focus cycle and hands off to a break, following the S-50 mandatory-break rhythm — it does not silently continue the cycle on another task
  - Blocked tasks stay out of the wedge suggestion pool until unblocked
  - User can move a blocked task back to active when the dependency clears
- **Unknowns:**
  - Does blocking mid-cycle reuse S-50's user-chosen break (short/long ★) or force a default break? — owner: user. Block: no.
  - Does a blocked task carry a "waiting on what" note (reuse of the S-18 resume note) or stay a bare state? — owner: user. Block: no.
  - Do blocked tasks age into the S-44 stale archive, or is blocked exempt from auto-archive like daily standing tasks? — owner: implementer. Block: no.
  - Does the daily recap / Podsumowanie report blocked tasks separately from done and partial? — owner: implementer. Block: no.
- **Risk:** Blocked is a lifecycle state, so it touches the task repository contract on both sides of the guest/authenticated split, the scorer's candidate pool, and the S-50 completion-to-break handoff at once — a guest-only or server-only implementation fails silently. The mid-cycle surface sits next to "Gotowe" inside the timer hub, which is high-blast-radius (`use-pomodoro-cycle` + `pomodoro-dashboard`) — run `pnpm change-impact` before editing.
- **Status:** ready
