# S-44: Stale task archive

- **Outcome:** user can review tasks older than 3 days in a separate archive view and permanently delete selected items via multi-select + delete selected
- **Change ID:** archive-old-tasks
- **Linear:** [FLO-94](https://linear.app/flowstate-10xdev/issue/FLO-94/flowstate-stale-task-archive-view-bulk-delete-s-44)
- **GitHub:** [#178](https://github.com/konrad-kaluzny-ceneo/FlowState/issues/178)
- **PRD refs:** US-05
- **Prerequisites:** S-09, S-13
- **Parallel with:** S-40 (soft — archive entry placement in home IA)
- **Blockers:** —
- **Acceptance:**
  - Active tasks with no touch for 3+ days automatically leave the default task list and appear in a dedicated archive view
  - Archive is a separate view reachable from home/task inventory — not mixed into the active list
  - User can select multiple archived tasks and permanently delete them in one action
  - Permanent delete removes tasks from the archive list; active list and wedge suggestion pool stay unchanged for remaining tasks
- **Resolved decisions:**
  - Age anchor — `updatedAt ?? createdAt`; a task is stale after 3+ days without a touch.
  - Daily standing tasks — exempt from auto-archive.
  - Archive view entry placement — implemented ahead of S-40 home IA reset.
- **Risk:** Auto-archive could hide tasks the user still intends to work on; archive must be discoverable and bulk delete must be explicit — not confused with mark-done completion.
- **Status:** in review
