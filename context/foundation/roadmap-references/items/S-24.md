# S-24

### S-24: Cycle pause and resume

- **Outcome:** user can temporarily pause a running work or break cycle and resume later with remaining time preserved — without ending the cycle, without marking INTERRUPTED, and without incrementing session `interruptionCount`. **While PAUSED, all wedge gates are suppressed** (check-in, kickoff, suggestion, wind-down — pol-12). After **~30 min pause cap**, session ends calmly with closure (pol-8 / `Session Auto-Ended After Pause Cap`).
- **Change ID:** cycle-pause-resume
- **Linear:** [FLO-39](https://linear.app/flowstate-10xdev/issue/FLO-39)
- **GitHub:** [#50](https://github.com/konrad-kaluzny-ceneo/FlowState/issues/50)
- **PRD refs:** US-04 (PRD v3), FR-019, NFR (timer drift ≤ ±2s), guardrail no silent loss
- **Prerequisites:** S-01, S-02; **F-07 recommended** so conductor can suppress gates while `CycleState.PAUSED`
- **Parallel with:** S-09, S-22
- **Blockers:** —
- **PRD v3 decisions (2026-06-13):**
  - Pause does **not** count as interruption for scoring.
  - Pause does **not** extend 4h wall-clock inactivity — separate **~30 min pause cap** then auto-end session calmly.
  - Exact cap tunable (~30 min default); owner: user. Block: no.
- **Event storming decisions (2026-06-18):**
  - **Hybrid model:** domain events `Cycle Paused` / `Cycle Resumed` + `CycleState.PAUSED` + `pausedAt` + `remainingDurationSec` in schema — not overload `INTERRUPTED`.
  - **pol-12:** Conductor / hook must not surface wedge overlays while cycle is PAUSED.
  - **pol-8:** Cap exceeded → calm session end (closure line), not raw interrupt.
  - Distinct from **Interrupt** (abandon timer, dead-end) — see [`user-flow.md`](../../user-flow.md) §4 mid-cycle intents.
- **Unknowns:**
  - Server-side `PAUSED` + `remainingMs` vs client-only timer freeze? Owner: implementer. Block: no.
- **Risk:** PAUSED semantics touch Prisma enum, tRPC, guest blob, timer worker, refresh recovery — half-implemented pause worse than today's Interrupt.
- **Status:** done
